# 导航站需求与功能说明

## 项目概述

构建一个静态发布的导航页，作为个人站点群的入口中心。该导航页需要能导航到各个子站点，同时具备公告发布能力（Markdown 文档），用于发布网站通知公告、更新日志等内容。

## 技术栈

- **Astro** v6 — 静态站点生成器
- **Tailwind CSS** v4 — 原子化 CSS 框架
- **TypeScript** — 类型安全

## 功能需求

### 1. 子站点导航
- 首页展示各子站点的卡片式入口
- 支持一键跳转到外部子站点
- 清晰展示各站点的名称与功能描述
- **当前首页仅展示子站点与文档站**（工具站/资源站因与聚合工具定位重复，暂不展示）

### 2. 公告发布能力
- 基于 Markdown 编写内容
- 支持 Frontmatter 元数据（标题、描述、日期、分类等）
- 文档分类：通知公告、更新日志、使用指南
- **公告中心页面区分展示**：通知公告 与 更新日志 两大区块
- **首页展示最新 2 篇通知公告** + **第三方站点导航（带点击记忆前置）** + **大版本更新日志时间轴**
- **更新日志专页** (`/changelog`)：中间竖向时间轴，大版本/活动更新在左，小更新在右
- **草稿箱发布工作流**：将 `.md` 文件放入 `drafts/` 目录，运行 `npm run publish` 即可自动发布

### 3. 响应式设计
- 适配桌面端与移动端
- 统一的导航栏与页脚布局

### 4. 工具/小游戏聚合（预留扩展）
- 纯前端实现，无需后端交互
- 当前为预留页面，未在导航栏和首页展示
- 未来可能扩展：Base64 编解码、JSON 格式化、密码生成器、颜色转换器、贪吃蛇、2048 等

## 使用方式

### 发布公告 / 更新日志

**方式一：草稿箱发布（推荐）**

1. 在 `drafts/` 目录下创建 Markdown 文件
2. 文件第一行写 `# 标题`，脚本会自动提取
3. 运行 `npm run publish`
4. 脚本自动补齐 frontmatter、移动到 `src/content/docs/`，并可选立即构建

**更新日志 `level` 字段：**

仅 `category: changelog` 时生效：
- `major` — 大版本更新，显示在 `/changelog` 时间轴左侧，首页时间轴也会展示
- `event` — 活动更新，显示在 `/changelog` 时间轴左侧
- `minor` — 小更新，显示在 `/changelog` 时间轴右侧，首页不展示

**方式二：交互式创建**

```bash
npm run new
```

按提示选择类型、输入标题，自动生成带 frontmatter 的空白模板。

**Frontmatter 字段：**

```yaml
---
title: 标题                        # 必填
description: 简介                  # 可选
date: 2026-05-01                   # 必填
category: announcement             # announcement | changelog | guide
level: minor                       # minor | major | event（仅 changelog 生效）
draft: false                       # true 则隐藏
---
```
