# 项目结构说明

## 目录树

```
apparent-apogee/
├── astro.config.mjs              # Astro 配置文件（集成 Tailwind CSS vite 插件）
├── content.config.ts             # Astro v6 内容集合配置
├── package.json                  # 项目依赖与脚本
├── tsconfig.json                 # TypeScript 配置
├── drafts/                       # 草稿箱：放置待发布的 Markdown 文件
│   └── README.md                 # 草稿箱使用说明
├── scripts/                      # 辅助脚本
│   ├── new-post.js               # 交互式创建空白公告/更新日志模板
│   └── publish.js                # 扫描 drafts/ 并发布到内容集合
├── public/                       # 静态资源目录
│   ├── favicon.svg               # 网站图标
│   └── images/                   # 图片资源目录
│       ├── backgrounds/          # 页面背景图
│       ├── logo/                 # 网站 logo
│       ├── illustrations/        # 插图、横幅
│       └── avatars/              # 头像
├── src/
│   ├── content/
│   │   └── docs/                 # Markdown 文档源文件（公告与更新日志）
│   │       ├── site-announcement.md   # 网站上线公告
│   │       ├── site-rules.md          # 网站规范与使用协议
│   │       ├── changelog-v1.md        # v1.0.0 更新日志
│   │       └── how-to-use.md          # 使用指南
│   ├── layouts/
│   │   └── Layout.astro          # 全局页面布局（导航栏 + 页脚 + HTML 骨架）
│   ├── pages/                    # 文件系统路由
│   │   ├── index.astro           # 首页：子站点导航 + 最新公告 + 更新日志时间轴
│   │   ├── docs/
│   │   │   ├── index.astro       # 公告中心（区分通知公告与更新日志）
│   │   │   └── [...slug].astro   # 文档详情页（动态路由，渲染 Markdown）
│   │   └── tools/
│   │       └── index.astro       # 工具/小游戏聚合入口页（预留）
│   └── styles/
│       └── global.css            # 全局样式：Tailwind CSS + 自定义 prose 样式
└── dist/                         # 构建输出目录（静态 HTML）
```

## 各文件详细说明

### 配置文件

| 文件 | 说明 |
|------|------|
| `astro.config.mjs` | Astro 主配置，通过 Vite 插件集成 Tailwind CSS v4 |
| `content.config.ts` | 定义 `docs` 内容集合的 loader（glob 模式加载 `src/content/docs/*.md`）和 schema（title/description/date/category/draft） |
| `tsconfig.json` | 继承 `astro/tsconfigs/strict`，开启严格类型检查 |

### 辅助脚本

| 文件 | 说明 |
|------|------|
| `scripts/new-post.js` | 交互式创建空白 Markdown 文件。运行 `npm run new` 可交互选择类型、输入标题，自动生成带 frontmatter 的模板文件到 `src/content/docs/` |
| `scripts/publish.js` | 发布草稿。运行 `npm run publish` 扫描 `drafts/` 目录，自动补齐 frontmatter、生成文件名、移动到 `src/content/docs/`，并可选立即构建 |

### 布局与样式

| 文件 | 说明 |
|------|------|
| `src/layouts/Layout.astro` | 全局布局，包含：`<html>` 骨架、`<meta>` SEO 标签、顶部导航栏（首页/公告两个入口，当前页面高亮）、页脚版权信息 |
| `src/styles/global.css` | 导入 Tailwind CSS；定义自定义主题色（primary/surface/text/border）；为 Markdown 渲染的 `.prose` 内容定制样式（标题、代码块、引用、列表等） |

### 页面路由

| 路由 | 源文件 | 说明 |
|------|--------|------|
| `/` | `src/pages/index.astro` | 首页。展示子站点导航（含聚合工具）、最新 2 篇通知公告、第三方站点导航、大版本更新日志时间轴 |
| `/docs` | `src/pages/docs/index.astro` | 公告中心。分「通知公告」与「更新日志」两个独立区块展示 |
| `/docs/:slug` | `src/pages/docs/[...slug].astro` | 文档详情页。动态路由，读取 Markdown 内容并渲染，顶部显示分类标签与发布日期，底部有返回链接 |
| `/changelog` | `src/pages/changelog.astro` | 更新日志专页。竖向时间轴布局，大版本/活动更新在左，小更新在右 |
| `/tools` | `src/pages/tools/index.astro` | 工具聚合入口（预留）。当前未在导航栏和首页展示 |

### 内容集合

| 目录 | 说明 |
|------|------|
| `src/content/docs/` | Markdown 文档存放目录。每篇文档需包含 Frontmatter，支持标准 Markdown 语法 |
| `drafts/` | 草稿箱。将写好的 Markdown 文件放入此处，运行 `npm run publish` 即可自动发布到 `src/content/docs/` |

**Frontmatter 字段：**

```yaml
---
title: 文档标题                    # 必填
description: 文档简介              # 可选，用于列表页摘要
date: 2026-05-01                   # 必填，发布日期
category: announcement             # 可选，默认 announcement。可选值：announcement | changelog | guide
level: minor                       # 可选，默认 minor。仅 changelog 生效：minor | major | event
draft: false                       # 可选，默认 false。设为 true 则不在列表中显示
---
```

## 当前构建产物

执行 `npm run build` 后，`dist/` 目录生成以下静态页面：

```
dist/
├── index.html                    # 首页
├── docs/
│   ├── index.html                # 公告中心
│   ├── site-announcement/
│   │   └── index.html            # 公告详情
│   ├── site-rules/
│   │   └── index.html            # 规范详情
│   ├── changelog-v1/
│   │   └── index.html            # 更新日志详情
│   └── how-to-use/
│       └── index.html            # 使用指南详情
└── tools/
    └── index.html                # 工具聚合入口
```

所有页面均为纯静态 HTML，可直接部署到任意静态托管服务（如 Vercel、Cloudflare Pages、GitHub Pages、Nginx 等）。

## 扩展指南

### 发布公告 / 更新日志（推荐方式）

1. 将写好的 Markdown 文件放入 `drafts/` 目录
2. 文件第一行写 `# 标题` 即可，脚本会自动提取
3. 运行 `npm run publish`
4. 脚本自动补齐 frontmatter、生成文件名、移动到 `src/content/docs/`
5. 确认构建后部署 `dist/` 目录

如需手动指定 category 或 date，可在草稿文件顶部手写 frontmatter：

```markdown
---
title: 标题
date: 2026-05-01
category: changelog
---

正文...
```

### 交互式创建空白模板

```bash
npm run new
```

按提示选择类型、输入标题，即可在 `src/content/docs/` 下生成带 frontmatter 的空白文件。

### 添加第三方站点

编辑 `src/pages/index.astro`，在 `thirdPartySites` 数组中追加新对象：

```astro
{
  name: '站点名称',
  url: 'https://example.com',
  icon: '🚀',
  color: 'bg-purple-50 border-purple-200 hover:border-purple-400',
  iconBg: 'bg-purple-100 text-purple-600',
}
```

第三方站点支持浏览器本地记录（localStorage）：用户点击后会自动记住，下次访问时该站点会被前置到第一位。

### 添加新子站点导航

编辑 `src/pages/index.astro`，在 `externalSites` 数组中追加新对象：

```astro
{
  title: '新站点',
  url: 'https://example.com',
  description: '站点描述',
  icon: '🚀',
  color: 'bg-purple-50 border-purple-200 hover:border-purple-400',
  iconBg: 'bg-purple-100 text-purple-600',
}
```

### 添加图片资源

将图片文件放入 `public/images/` 下的对应目录，构建时会自动复制到 `dist/`：

| 目录 | 用途 | 推荐路径 |
|------|------|----------|
| `backgrounds/` | 页面背景图 | `/images/backgrounds/hero-bg.jpg` |
| `logo/` | 网站 logo | `/images/logo/site-logo.png` |
| `illustrations/` | 插图、横幅 | `/images/illustrations/banner.png` |
| `avatars/` | 头像 | `/images/avatars/site-avatar.png` |

在页面中使用：

```astro
<img src="/images/illustrations/banner.png" alt="横幅" />
```

在 CSS 中使用背景图：

```css
.my-section {
  background-image: var(--hero-bg, none);
}
```

### 添加新工具/小游戏

1. 创建新页面，如 `src/pages/tools/base64.astro`
2. 在 `src/pages/tools/index.astro` 中将对应卡片的 `href` 改为实际路径，`status` 改为 `done`
3. 移除卡片右上角的「规划中」标签（可选）
